## Problem
Robustness follow-ups from the Epic 4 reviews (4-5 / 4-2). (1) In src/shell/loop.ts the onModeChange (and draw / sampleInput) callbacks are invoked unguarded — if a callback throws, the requestAnimationFrame chain dies and the game silently halts. Now that 4-6 registers a real onModeChange (save-on-commit), wrap the callback invocation so a throw cannot stop the loop (try/catch in one place, or a documented and enforced "callbacks must not throw" contract). (2) In the select-mode handler in src/core/sim.ts, input.spin is not finite-guarded — a NaN spin passes the !== 0 check and poisons selectedLevel via Math.sign(NaN); guard non-finite spin. (3) Remove the dead s.spikes[2]=0.5 setup line in tests/core/sim.death.test.ts (its assertion was already removed). Context: sprint/archive/4-5-session.md Reviewer Assessment (unguarded callback) and sprint/archive/4-2-session.md Reviewer Assessment (L2 NaN spin, dead test line).
