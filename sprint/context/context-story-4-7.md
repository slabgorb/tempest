## Problem

The shell/render.ts currently has no rendering branches for the framing screens (attract, select, highscore entry, gameover). On boot and after game-over→attract transitions, the renderer draws a stale playing-scene frame (tube + ghost enemies) instead of the appropriate framing screen. Additionally, lives are displayed but not yet as Claw-icon glyphs, and the HUD lacks the high-score display.

**Reference:** 4-2 finding F1: "render.ts currently has NO attract/select branch, so on boot and after gameover→attract the renderer draws a stale playing frame."

## Technical Approach

**Scope:** Shell rendering only (`src/shell/render.ts`, possibly `src/main.ts` wiring). The pure core is already complete (4-2 modes, 4-3 highscore state machine, 4-5 onModeChange hook). 4-7 only visualizes the existing game state.

1. **Attract screen** (mode `'attract'`): Draw title, high-score table, and "press start" prompt. Suppress stale playing scene.
2. **Select screen** (mode `'select'`): Show `state.select.selectedLevel` with instructions for changing/confirming it.
3. **Highscore entry screen** (mode `'highscore'`): Render the initials entry UI showing current letter and position (`state.entry.initials`, `state.entry.charIndex`, `state.entry.currentLetter`).
4. **Gameover screen** (mode `'gameover'`): Enhance existing gameover rendering with score/high-score table display.
5. **Lives HUD:** Replace numeric lives display with small Claw icons during play.
6. **Score/high-score HUD:** Add high-score display alongside score.

**Architecture:** Add a mode-dispatch switch early in `render()` to branch before drawing the game scene. For non-playing modes (attract/select/highscore/gameover), draw the appropriate framing UI only. Ensure the Claw icon glyph is reusable (it already exists in `drawPlayer` and `drawWarp`).

**Testing:** This project does NOT unit-test the shell/renderer. Verification via:
- `npx tsc --noEmit` (clean typecheck)
- `npm run build` (succeeds)
- Optional: quick Playwright screenshot of the dev server to confirm screens render without console errors.

## Scope
- In scope: attract/select/highscore/gameover screen rendering; lives as Claw icons; high-score HUD display.
- Out of scope: core game logic changes; input handling (4-5 onModeChange already wired); persistence (4-6 wires it).

1. **Attract screen renders correctly:** Title, high-score table, "press start" prompt. Stale game scene is suppressed (F1 requirement).
